MÓDULO 6 · LECCIÓN 2 · ACTIVIDAD

Templates y Rendering en Vue

Ficha de actividades para el alumno · Composition API (<script setup>)

🕒 Duración estimada: 90–120 min 🧩 6 actividades + 1 integradora 💻 Modalidad: individual

📋 Antes de empezar — lee esto

El objetivo de esta actividad NO es entregar código perfecto. Es que te pelees con cada concepto y entiendas el por qué. Si copias y pegas sin entender, perdiste el tiempo. Si te trabas, primero piensa, después revisa tus apuntes, después usa las pistas, y recién al final pregunta.

🛠️ Setup del proyecto

  • Usa el proyecto Vite que creaste en la Lección 1 (o crea uno nuevo con npm create vue@latest).
  • Crea una carpeta src/components/actividades/ y adentro un componente .vue por cada actividad (ej: Actividad1.vue).
  • Importa y muestra el componente que estés trabajando desde App.vue.
  • TODOS los componentes deben usar <script setup>. Nada de Options API acá.
Regla de oro que no puedes olvidar: en el <script> accedes al valor de un ref con .value. En el <template>, NUNCA pongas .value. Si te da [object Object] o undefined, ese suele ser el problema.

1 Tarjeta de perfil Fácil

🎯 Meta: dominar interpolaciones, ref y v-bind de atributos.

Crea un componente TarjetaPerfil.vue que muestre los datos de una persona (inventa los tuyos).

Requisitos

  • Declara al menos 4 datos con ref: nombre, profesion, edad y urlFoto.
  • Muestra nombre, profesión y edad usando interpolación {{ }}.
  • Muestra la foto con una etiqueta <img> enlazando el src de forma dinámica.
  • Agrega un texto que diga "Mayor de edad: SÍ / NO" calculado con una expresión en el template.
💡 Pista: los mustaches {{ }} NO funcionan dentro de atributos HTML. Para el src de la imagen necesitas otra directiva. ¿Cuál era la de atributos? Para el "mayor de edad", piensa en un operador ternario.
Ver pista extra (solo si te trabaste de verdad)

El atributo dinámico se hace con :src="urlFoto" (forma abreviada de v-bind:src). Para la mayoría de edad: {{ edad >= 18 ? 'SÍ' : 'NO' }}. Puedes usar https://i.pravatar.cc/150 como foto de prueba.

2 Contador con reglas Fácil

🎯 Meta: eventos con @click, funciones manejadoras y renderización condicional.

Crea ContadorReglas.vue: un contador que arranca en 0 con botones para sumar, restar y resetear.

Requisitos

  • Un ref contador iniciado en 0.
  • Tres botones: +1, −1 y Reset, cada uno con su función manejadora.
  • El contador NO puede bajar de 0 (si está en 0, restar no hace nada).
  • Muestra un mensaje con v-if/v-else-if/v-else: "En cero" (0), "Vas bien" (1–9), "¡Nivel experto!" (10+).
💡 Pista: en Composition API una "función manejadora" es una function normal en el <script setup>. Recuerda el .value para leer y modificar el ref dentro de esa función. Para el tope inferior, un simple if dentro de la función de restar.

3 Formulario reactivo Medio

🎯 Meta: v-model, modificadores y @submit.prevent.

Crea FormularioSaludo.vue: un input donde el usuario escribe su nombre y ves el resultado en vivo.

Requisitos

  • Un input enlazado con v-model a un ref nombre.
  • Debajo del input, un saludo en vivo: "Hola, {{ nombre }} 👋" que se actualiza al escribir.
  • Usa el modificador que elimina espacios sobrantes al inicio y fin.
  • Un formulario con botón "Enviar" que al enviarse NO recargue la página y muestre un alert con el nombre.
  • Si el input está vacío, muestra el texto "Escribe tu nombre para ver el saludo" en lugar del saludo.
💡 Pista: el modificador para limpiar espacios se pega al v-model con un punto. Para evitar la recarga, hay un modificador de eventos que llama a preventDefault() por ti. Para el mensaje vacío, combina v-if / v-else revisando la longitud del texto.
Ver pista extra

v-model.trim="nombre" · @submit.prevent="miFuncion" · condición del vacío: v-if="nombre.length === 0".

4 Toggle de detalles: v-if vs v-show Medio

🎯 Meta: entender la diferencia REAL entre v-if y v-show.

Crea DemoToggle.vue con dos bloques de texto y un botón que los muestra/oculta.

Requisitos

  • Un ref booleano visible y un botón que lo alterna.
  • Un párrafo controlado con v-if="visible".
  • Otro párrafo controlado con v-show="visible".
  • Abre las DevTools (F12), inspecciona el DOM y observa qué pasa con cada uno al ocultarlos.
  • Escribe en un comentario dentro del componente: ¿cuál desaparece del DOM y cuál solo se oculta? ¿Cuándo usarías cada uno?
⚠️ Esta actividad no se trata de código, se trata de ENTENDER. Si no abres las DevTools y no ves la diferencia con tus propios ojos, no cumpliste la actividad. La reflexión escrita es obligatoria.

5 Lista de productos con v-for Medio

🎯 Meta: renderizar listas con v-for, :key y alias de índice.

Crea ListaProductos.vue que muestre un catálogo a partir de un arreglo.

Requisitos

  • Un ref productos = arreglo de al menos 4 objetos con id, nombre y precio.
  • Renderiza la lista con v-for, mostrando índice + nombre + precio.
  • Usa :key correctamente (piensa qué valor va ahí).
  • Si un producto cuesta más de $10.000, muestra una etiqueta "⭐ Premium" al lado (con v-if).
  • Agrega un mensaje de empty state ("No hay productos") que aparezca si el arreglo está vacío. Pruébalo vaciando el arreglo.
💡 Pista: el :key debe ser único por elemento — el id es el candidato perfecto, NO uses el índice. Para el índice como alias: v-for="(prod, i) in productos". El empty state se resuelve mirando productos.length.
Ver pista extra

Estructura: <li v-for="(prod, i) in productos" :key="prod.id">. La etiqueta premium: <span v-if="prod.precio > 10000">⭐ Premium</span>. Empty state: un v-if="productos.length === 0" arriba de la lista.

6 Ficha dinámica desde un objeto Reto

🎯 Meta: iterar las propiedades de un objeto con v-for (clave + valor).

Crea FichaObjeto.vue que reciba un objeto y liste TODAS sus propiedades automáticamente, sin escribirlas a mano.

Requisitos

  • Un ref personaje = objeto con al menos 5 propiedades (ej: nombre, raza, poder, planeta, nivel).
  • Recorre el objeto con v-for mostrando "clave: valor" por cada propiedad.
  • El desafío: si mañana agregas una propiedad más al objeto, la ficha debe mostrarla sin tocar el template.
💡 Pista: v-for puede recorrer objetos y te da acceso a (valor, clave) en ese orden — ¡ojo con el orden, no es al revés! Igual necesitas un :key. ¿Qué valor único tienes en cada iteración de un objeto?

★ Actividad Integradora: Mini gestor de tareas Integradora

🎯 Meta: aplicar TODO lo de la lección en una sola pieza funcional — como una feature real.

Crea GestorTareas.vue. Es tu proyecto final de la lección: un administrador de tareas donde el usuario agrega, completa y elimina tareas. Acá se junta todo lo que practicaste.

Funcionalidades obligatorias

  • Agregar tarea: un input + botón (o formulario con @submit.prevent) que agrega una tarea nueva al arreglo. Usa v-model con el modificador de limpieza de espacios.
  • Validación: si el input está vacío, no se agrega nada.
  • Listar tareas: muéstralas con v-for y :key correcto. Cada tarea es un objeto con id, texto y completada.
  • Completar tarea: al hacer clic en una tarea, se marca como completada (y visualmente se ve distinta: tachada, otro color, etc. — usa enlace dinámico de clase o estilo).
  • Eliminar tarea: un botón ✕ en cada tarea que la quita de la lista.
  • Empty state: si no hay tareas, muestra un mensaje amigable con v-if/v-else.
  • Contador en vivo: muestra "Tareas pendientes: N" calculado en el template a partir del arreglo.

Retos opcionales (para el que quiere destacar) 🚀

  • Un botón que use v-show para ocultar/mostrar las tareas ya completadas.
  • Un mensaje "¡Todo listo! 🎉" que aparezca solo cuando no quedan tareas pendientes.
  • Mostrar la fecha/hora en que se creó cada tarea.
💡 Pistas de arquitectura (no la solución):
  • El estado principal es un ref con un arreglo de objetos. Para agregar: tareas.value.push(...). Para eliminar: piensa en .filter().
  • Necesitas una variable para generar id únicos (un contador que solo sube). No uses el índice como id.
  • Para "completar", pasa la tarea (o su id) a una función que invierta su propiedad completada.
  • El estilo dinámico de la tarea completada se hace enlazando :class o :style a una condición.
  • El contador de pendientes: una expresión con .filter() directamente en el {{ }}.
⚠️ No busques "todo app vue" en Google y copies. Vas a encontrar 500 tutoriales resueltos. Si copias, te engañas a ti mismo. El músculo se hace levantando el peso, no mirando cómo otro lo levanta. Pelea con el problema primero.

✅ Checklist antes de entregar

  • Todos los componentes usan <script setup> (Composition API).
  • No hay ningún .value escrito dentro del <template>.
  • Cada v-for tiene su :key único (y NO es el índice).
  • El código no tiene console.log ni comentarios de basura olvidados.
  • Probaste todos los casos: agregar, completar, eliminar, lista vacía.
  • Nombres de variables y funciones claros y en español o inglés consistente.
